464    /// 2. The service sends back a ProcessingResponse message that directs Envoy
465    ///    to either stop processing, continue without it, or send it the
466    ///    next chunk of the message body.

468    ///    or the entire body at once. In either case, the server sends back
469    ///    a ProcessingResponse after each message it receives.
470    /// 4. If so requested, Envoy sends the server the HTTP trailers,
471    ///    and the server sends back a ProcessingResponse.
472    /// 5. At this point, request processing is done, and we pick up again
473    ///    at step 1 when Envoy receives a response from the upstream server.

475    ///    then Envoy proceeds without consulting the server.

478    ///    was configured to ignore errors.
